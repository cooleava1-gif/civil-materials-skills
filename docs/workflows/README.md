# Workflow Demos

These demos are the guided front door for the bundle. They do not replace the
skill routers, but they let a new user see how the system is meant to be used
from the first prompt through the expected artifacts.

## Workflow Index

1. [WER-EA mini-review](wer-ea-mini-review.md)
2. [Experimental manuscript](experimental-manuscript.md)
3. [Revision loop](revision-loop.md)
4. [Paper to presentation](paper-to-presentation.md)

## How To Use These Demos

- Start with the route that matches the deliverable you want most.
- Use the demo prompt as your first real prompt inside Codex.
- Compare your actual outputs against the expected artifacts and the
  "What Good Looks Like" checklist.
- If your task drifts, return to `materials-research` and re-route before
  asking a downstream skill to improvise.

## Shared Rule

These demos are workflow patterns, not substitute evidence. The final
manuscript, figure, or response package still needs real source-grounded
content, bounded claims, and journal-aware review.
