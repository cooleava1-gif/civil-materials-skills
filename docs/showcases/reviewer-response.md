# Reviewer response

## Outcome Snapshot

This showcase is for the deliverable that goes back to editors and reviewers:
the point-by-point response package. A good outcome does not only sound polite;
it proves which manuscript actions were taken and which claims were bounded
instead of overpromised.

## Demo Prompt

```text
Take these reviewer comments and build a revision-ready response package with proof of change and routed weaknesses.
```

## Build Path

1. Start with `materials-response` once reviewer comments are available.
2. Route technical weaknesses back into reader, citation, writing, figure,
   polishing, or data work before finalizing the letter.
3. Use the example response files to keep the tone specific and evidence-backed.
4. Return to the response package only after the manuscript-side repairs are
   real enough to cite.

## When To Use This Route

Use this showcase when the core need is revision proof, not merely revised
manuscript prose. It is strongest for major or method-heavy revisions where the
letter has to show exactly what changed and what still remains bounded.
