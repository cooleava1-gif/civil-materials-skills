# Reviewer Response Patterns

Use this reference when the reviewer comment is common but the response strategy is not obvious. Each pattern separates what to revise, what to say, and what not to overpromise.

## 1. English Needs Major Revision

Strategy:

- acknowledge readability concern.
- state that the manuscript was fully edited.
- mention technical terminology consistency.
- do not claim professional editing unless it happened.

Example:

> We have thoroughly revised the manuscript for grammar, clarity, and technical consistency. In particular, the descriptions of bonding strength, curing condition, and moisture conditioning were rewritten to reduce ambiguity. The revised text can be found throughout the manuscript, especially in Sections [x-y].

## 2. Add More References

Strategy:

- classify requested references: review, primary mechanism, performance comparison, standards.
- add only relevant sources.
- explain where they were used.

Example:

> We added recent references on waterborne epoxy emulsified asphalt, tack coat bonding, and moisture-conditioned interlayer performance. These references are now used to clarify the research gap in the Introduction and to compare the pull-off strength results in Section [x].

## 3. Incremental or Local-Condition Novelty

Strategy:

- avoid defensive claims.
- reframe contribution as a bounded material-system/evidence combination.
- revise novelty wording if overbroad.

Example:

> We agree that the original novelty statement was too broad. The revised manuscript now positions the contribution as the combined evaluation of emulsion stability, curing behavior, bonding performance, and moisture-conditioned retention for this waterborne epoxy emulsified asphalt system, rather than as a universal modifier mechanism.

## 4. Literature Comparison Is Insufficient

Strategy:

- add a comparison table if standards/conditions are compatible.
- if not compatible, compare trends and boundary conditions rather than absolute values.

Example:

> We added Table [x] comparing epoxy type, dosage, curing condition, test method, and reported bonding strength from relevant studies. Because pull-off fixtures and curing protocols differ among studies, the comparison is used as contextual evidence rather than a direct ranking.

## 5. Sample Size Too Small

Strategy:

- state replicate count.
- add uncertainty if available.
- reduce claim strength if sample size cannot be increased.

Example:

> We now report the replicate count and SD for each group. We acknowledge that n = [x] limits statistical power; therefore, the conclusion has been revised from `significant improvement` to `higher mean strength under the tested condition` where statistical significance was not established.

## 6. Why Method A Not Method B

Strategy:

- explain method fit to research question.
- acknowledge what method B would add.
- add limitation/future work if needed.

Example:

> The pull-off test was selected because the study focuses on tensile interlayer adhesion under tack-coat application conditions. We agree that shear testing provides complementary information; this limitation is now stated in Section [x], and the conclusion is limited to tensile bond performance.

## 7. Add Error Bars or Report SD

Strategy:

- add error bars and define SD/SE/CI.
- report replicate count in caption.
- if raw data are unavailable, state limitation and avoid statistical claims.

Example:

> Error bars representing SD have been added to Figures [x-y], and the replicate count is now included in each caption. The Methods section also states how repeated specimens were prepared and averaged.

## 8. More Suitable for Another Journal

Strategy:

- respond through scope, not prestige.
- strengthen journal-fit language.
- if editor decision is firm, do not argue excessively.

Example:

> We have revised the Introduction and Conclusion to emphasize the construction-materials relevance of the work, including tack coat application, interlayer bonding, and moisture-conditioned performance. We believe these revisions clarify the fit with the journal's scope in construction materials.

## 9. Provide Raw Data as Supplementary

Strategy:

- provide raw/processed data package when possible.
- include metadata and test conditions.
- if constrained, provide available subset and explain limits.

Example:

> We prepared a supplementary dataset containing raw measurements, processed averages, replicate counts, units, and test conditions. The data availability statement has been updated accordingly.

## 10. Novelty Is Not Justified

Strategy:

- avoid claiming `first`.
- map what is new: material system, evidence combination, service condition, or framework.
- revise Introduction gap and Conclusion.

Example:

> The novelty statement has been revised to focus on the simultaneous assessment of curing, storage stability, bonding performance, and moisture-conditioned retention. We also added recent studies to clarify how the present evidence chain differs from previous performance-only reports.

## 11. Conflicting Reviewer Requests

Strategy:

- identify shared concern behind both comments.
- avoid saying one reviewer is wrong.
- make a balanced revision.

Example:

> Reviewer 1 requested a more detailed mechanism discussion, whereas Reviewer 2 cautioned against speculative interpretation. We addressed both concerns by adding the available FTIR/microscopy evidence while revising the mechanism wording to distinguish direct observations from interpretation.

## 12. Requested Experiment Is Beyond Scope

Strategy:

- acknowledge value of experiment.
- explain practical/scope limitation.
- reduce claim strength and add future work.
- do not use lack of equipment as a dismissive excuse.

Example:

> We acknowledge that additional [test] would strengthen the mechanism evidence. Because the present study was designed around [available tests/scope], this experiment is beyond the current scope. We have therefore revised the mechanism discussion to be more cautious and added [test] as future work.
