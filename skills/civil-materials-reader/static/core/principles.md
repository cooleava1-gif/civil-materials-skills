# Reader Contract

The reader must be source-grounded.

Required sections:

- Bibliographic identity.
- Research problem.
- Material system.
- Experiment matrix.
- Figure/table evidence map.
- Key results.
- Mechanism chain.
- Limitations.
- Relevance to the user's project.

For WER-EA intensive reading packages, required files are:

- `paper.md`
- `source_map.json`
- `translation_notes.md`
- `figure_table_cards.md`
- `mechanism_evidence_table.md`
- `dosage_window_table.md`
- `review_handoff.md`
- `obsidian_note.md`
- `visual_asset_spec.json` when a source PDF is available
- `assets/README.md`
- `assets/asset_manifest.md`, `assets/contact_sheet.png`, rendered pages, and cropped assets when visual inspection is claimed

Do not invent missing data, DOI, figure details, standards, or mechanisms.

Use flags:

- `[source missing]`
- `[figure not provided]`
- `[number not visible]`
- `[mechanism inferred, not directly proven]`
