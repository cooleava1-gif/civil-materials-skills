# Review Talk Deck

For review, perspective, or synthesis papers that organise existing knowledge rather than report new primary data.

## Presentation arc — evidence-map

The contribution is organisation and synthesis. Order the story as:

1. why the topic matters now,
2. the conceptual framework or classification system,
3. theme 1 with supporting evidence,
4. theme 2 with supporting evidence,
5. theme 3 with supporting evidence,
6. controversy or unresolved problem,
7. the author's synthesis or positioning,
8. future directions and research gaps.

## Default slide structure (adapt, do not force)

1. 标题页
2. 为什么这个主题现在重要（工程背景 + 文献增长趋势）
3. 概念框架 / 分类体系 / 全局地图
4-6. 主题 1-3（每个主题：关键发现 + 代表性数据 + 证据来源）
7. 争议 / 未解决的问题 / 研究空白
8. 作者的综合观点 / 定位
9. 未来研究方向
10. 总结

## Domain-specific guidance

### WER-EA / modified emulsified asphalt reviews

- Organize by material system (WER, SBR, SBS, crumb rubber) or by performance dimension (bonding, rheology, durability, mechanism).
- Include a dosage-comparison table as a reference slide.
- Show mechanism evidence chain: claim → evidence → mechanism → boundary.
- Flag where direct evidence is missing (only performance data, no characterization).

### Cement/concrete reviews

- Organize by material type, durability mechanism, or test method.
- Include a mix-proportions or test-conditions summary table.
- Show durability indicators (chloride diffusion, carbonation depth, freeze-thaw cycles) as comparison panels.
- Flag where field data is limited (mostly lab-scale studies).

### General rules

- A review deck leans more text- and schematic-led than a primary-research deck, but avoid dense bullet pages.
- Use a conceptual framework diagram as the spine of the presentation.
- Give each theme one clear visual or schematic.
- Attribute key ideas to their original sources, not to the review alone.
- Use a literature matrix or comparison table to show evidence coverage.
- End with how the review supports the user's own research project.
