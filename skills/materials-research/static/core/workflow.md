# Core Workflow

Use this workflow for all materials tasks.

1. Route the request by task, domain, and journal.
2. Restate the research object and intended deliverable in one sentence.
3. Build or inspect the claim-evidence ladder.
4. Identify the engineering context: pavement, building, repair, durability, sustainability, or transport infrastructure.
5. Apply the domain fragment to check tests, mechanisms, and missing controls.
6. Apply the journal fragment to check scope fit and reviewer expectations.
7. Produce the requested artifact plus a short "missing inputs or risks" note.

For data-heavy requests, add one extra step before writing: map each dataset to a figure/table and each figure/table to one claim.

If the user asks for a finished manuscript section, write the section first, then add a compact audit note.

If the user asks for planning, give a decision table, not vague advice.

If the user asks for current journal facts, verify official sources before making final claims.
