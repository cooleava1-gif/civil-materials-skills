# From Data to Manuscript

Use when the user provides tables, spreadsheet values, result descriptions, or figure drafts.

## Data-to-claim pipeline

1. Clean meaning before language: identify what each variable actually measures.
2. Compare against the control before comparing modified groups.
3. Look for trade-offs, not only maxima.
4. Decide the figure before writing the paragraph.
5. Write one claim per figure.
6. Add mechanism only when characterization supports it.

## Common mappings

| Data pattern | Figure/table | Manuscript claim |
|---|---|---|
| Modifier dosage vs bonding strength | grouped bar or line plot | optimum dosage and performance gain |
| Dry vs wet bonding | retention ratio plot | moisture resistance or water sensitivity |
| Viscosity/storage stability vs modifier | dual-axis or trade-off plot | constructability-performance balance |
| FTIR peak changes | annotated spectra | possible chemical interaction or curing evidence |
| SEM/fluorescence morphology | multi-panel image plus descriptor table | phase distribution or interface morphology |
| Multiple performance indexes | decision matrix | balanced formulation selection |

## Result paragraph skeleton

`Compared with [control], [modified group] showed [trend] under [test condition]. The maximum/optimum value occurred at [dosage], indicating [performance interpretation]. However, [trade-off or limitation] suggests that [bounded conclusion]. This trend is consistent with [mechanism evidence], but [missing evidence] should be addressed before making a stronger mechanism claim.`

## Reviewer-safe language

Use "showed", "indicated", "suggested", "was consistent with", and "may be associated with".

Avoid "proved", "completely", "significantly" without statistics, and "best" without a defined criterion.
