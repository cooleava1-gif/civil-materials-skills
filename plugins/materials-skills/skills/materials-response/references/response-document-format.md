# Reviewer Response Document Format

Use this reference when preparing a real revision package.

## Required Files

Most revision packages need:

- `Author Response` or `Response to Reviewers`: point-by-point replies.
- `Revised Manuscript with Tracked Changes`: all edits visible.
- `Clean Revised Manuscript`: if requested by journal.
- `Revision Cover Letter`: concise summary to editor.
- Supplementary data, raw data, or additional figures if promised in replies.

## Point-by-Point Format

Recommended structure:

```text
Reviewer 1, Comment 1:
[quote or paraphrase reviewer comment in italic/gray text]

Response:
We [acknowledge/agree/clarify] ...

Revision made:
The manuscript was revised on Page X, Lines Y-Z: "[brief changed text]"
```

Formatting:

- reviewer comments: italic, gray, or quoted block.
- author replies: normal font.
- manuscript changes: page/line or section references.
- use consistent numbering: `Reviewer 1, Comment 1`.

## Revision Cover Letter

Template:

```text
Dear Editor,

We appreciate the opportunity to revise our manuscript entitled "[title]".
In response to the reviewers' comments, we have [main revision 1],
[main revision 2], and [main revision 3]. A detailed point-by-point response
is provided below, and all changes are marked in the revised manuscript.

Sincerely,
[authors]
```

## Conflicting Reviewer Requests

See [response-patterns.md](response-patterns.md) Pattern 11 for the strategy, diplomatic wording, and a ready-to-use example response.

Key format rule: in the response document, present both reviewer concerns side by side, then state the single compromise action with manuscript locations for each reviewer's satisfaction.

## When Requested Experiments Are Beyond Scope

See [response-patterns.md](response-patterns.md) Pattern 12 for the strategy and example response.

Key format rule: always add the limitation to the manuscript's Limitations or Future Work section and point the reviewer to the exact location in the revised manuscript.

Never promise an experiment unless it has been completed or the user explicitly chooses that revision path.
