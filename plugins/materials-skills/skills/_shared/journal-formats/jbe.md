# Journal of Building Engineering (JBE) — Formatting Facts

Authoritative facts about JBE formatting requirements. This file holds the **facts**; each skill's `static/fragments/journal/jbe.md` adds the **action layer**.

> Always verify against the journal's current Guide for Authors before final submission. The numbers below are accurate as of early 2026.
> Official page: https://www.sciencedirect.com/journal/journal-of-building-engineering/publish/guide-for-authors

## Article types and limits

| Article type | Word limit | Abstract | References | Figures/Tables | Methods |
|---|---|---|---|---|---|
| **Research article** | No strict word limit (typically 6,000–10,000) | Structured (~250 words) | No hard cap (~50–80 typical) | No hard cap | Within main text |
| **Review article** | Longer (typically 8,000–15,000) | Unstructured (~300 words) | Comprehensive (100+) | Flexible | N/A |
| **Short communication** | ~3,000 | Structured (~150 words) | ~20 | ~4 | Within main text |

## Abstract

- Structured format for research articles
- ~250 words maximum
- No citations in abstract
- Emphasize building-engineering application and implications

## Figures and tables

| Parameter | Requirement |
|---|---|
| Resolution (halftone/photo) | 300 dpi minimum |
| Resolution (line art) | 1,000 dpi minimum |
| File formats | TIFF, EPS, PDF, or high-quality JPEG |
| Color mode | Color online |
| In-figure font | Legible at publication size |
| Panel labels | Lowercase letters in parentheses: (a), (b), (c) |

## References

- **Style**: Numbered, sequential by first appearance
- **In-text**: Square brackets [1], [2], [1,2], [1–5]
- **Format example**:
  ```
  [1] A.B. Smith, C.D. Johnson, Title of article, J. Build. Eng. 85 (2025) 108765.
  ```
- Journal names abbreviated per ISO 4
- Include DOI when available

## Mandatory statements

- **Data Availability statement** — required (Elsevier policy)
- **CRediT author statement** — required
- **Declaration of competing interest** — required

## JBE-specific requirements

- **Building-engineering application must be explicit** — materials contribution to building performance, durability, thermal/acoustic behavior, repair, or sustainability
- **Connect material data to building system implications** — not isolated material characterization

## Common desk-rejection patterns

1. **No building context** — pure material study without building-engineering application
2. **Road/pavement focus** — asphalt pavement topics are generally not JBE scope (route to CBM, RMPD, or IJPE)
3. **Missing building-system connection** — material data without implications for building performance
4. **Isolated lab optimization** — no connection to real building conditions or constraints
